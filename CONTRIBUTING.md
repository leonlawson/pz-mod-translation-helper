## 📝 翻译与校对贡献流程

### 方式一：通过可视化翻译工具（推荐）

这是推荐给大多数贡献者的方式，具体步骤请以 `README.md` 中的说明为准，典型流程如下：

1. 前往 [pz-mod-translation-helper Releases](https://github.com/LTian21/pz-mod-translation-helper/releases) 下载最新版本工具。
2. 解压并运行翻译工具程序。
3. 在工具中填写你的称呼和邮箱，并领取任务。
4. 在工具界面中完成翻译或校对。
5. 完成后点击“提交审核”，工具会自动创建 PR 或提交数据。

> 使用此方式提交，即表示你同意本项目的**版权与授权约定**（见下文），并且同意项目维护者将你的贡献整理后合并至本仓库。

### 方式二：直接通过 Git / Pull Request 提交

适合熟悉 Git 与 GitHub 的贡献者。整体步骤与 `README.md` 中“手动创建 Pull Request (高级选项)”一致，这里只简要列出：

1. Fork 本仓库并克隆到本地；
2. 基于 `main`（或指定分支）创建新分支：
   ```bash
   git checkout -b feature/your-translation

3. 编辑相关翻译文件（例如 `data/translations_CN.txt`），并按照规范：

   * 带前导 Tab 的条目为**待处理**；
   * 完成翻译或校对后，**删除行首所有 Tab 与前导空格**。
4. 提交并推送分支后，在 GitHub 上发起 Pull Request，并在 PR 描述中简要说明：

   * 翻译了哪些模组；
   * 主要修改内容（新增 / 校对 / 修错 等）。

## 🛠️ 脚本与工具开发贡献

本项目包含一些自动化脚本和工具（例如 `scripts/`、`translation_utils/` 目录），用于：

* 检查模组更新；
* 合并/导出翻译文件；

若你希望：

* 修复现有脚本中的 Bug；
* 为工具添加新特性；

可以按如下步骤：

1. Fork 本仓库并克隆到本地；
2. 基于最新分支创建新分支；
3. 在对应目录中修改或新增文件；
4. 提交前请尽量：

   * 保持原有代码风格；
   * 添加必要的注释；

5. 通过 PR 提交修改，并在描述中说明：
